03 — Per-Request Flow

Request Pipeline

Every API call passes through an ordered middleware chain. Each step can accept, reject, or modify the request before it reaches the upstream LLM provider.

TLS Termination + NGINX Ingress
All requests arrive over HTTPS. NGINX handles TLS termination, load balancing across gateway replicas, and basic rate limiting at the network edge.
NGINXTLS 1.3Kubernetes Ingress
Virtual Key Authentication
Extract Authorization: Bearer vk-xxxx header. SHA-256 hash the key and look up in Redis hot cache (fallback to PostgreSQL). Resolve to: provider, model allowlist, team, user, rate limits, DLP policy.
Cache hit <1ms404 → 401 UnauthorizedRevoked → 401Redis
Rate Limit Check
Check tokens-per-minute and requests-per-minute against sliding window counters in Redis. Per-key, per-user, and per-team limits are all enforced independently.
Exceeded → 429 Too Many RequestsSliding windowRedis INCR
DLP Scan (Data Loss Prevention)
Send prompt text to the Python DLP microservice (Microsoft Presidio). Detect PII, credentials, financial data, and custom org-defined patterns. Policy lookup determines action: BLOCK, REDACT, ALERT, or ALLOW.
BLOCK → 400 + Kafka alertREDACT → scrub + continueALERT → continue + logPresidio<5ms target
OPA Policy Evaluation
Evaluate declarative Rego policies: allowed models, time-of-day restrictions, geographic origin, content classifications, required prompt prefixes. All policies are versioned and auditable.
DENY → 403 ForbiddenALLOW → continueOPARego policies
Budget Check
Query current budget utilization for the team/user/project. Estimate token cost of the incoming request. Hard limit: reject if budget is exhausted. Soft limit: continue but send alert via BullMQ to notification queue.
Hard limit → 402 Payment RequiredSoft limit → alert + continuePostgreSQL + BullMQ
Smart Router (V3)
Select the optimal provider and model based on: real-time latency, cost per token, model capability tier, fallback chains, and optional A/B split configuration. Can substitute cheaper models for eligible request types.
Cost-optimized routingLatency-basedFallback chainsA/B splits
Provider Proxy + Response Streaming
Forward the request to the selected provider using the real API key retrieved from HashiCorp Vault. Stream SSE tokens back to the client in real time. Intercept the final usage object from the response.
SSE streamingVault key injectionUsage extraction
Async: Kafka Emit + Cost Aggregation
After response: emit cost event to Kafka (non-blocking). Workers consume and write to ClickHouse (analytics) and increment PostgreSQL budget counters. Prompt + response archived to S3. DLP alerts sent to OpenSearch.
KafkaClickHouseS3 archiveNon-blocking
04 — Versioned Roadmap

V1 → V4 Roadmap

Four release phases taking the platform from foundational observability to intelligent AI cost optimization, each building on the last.

V1 Foundation — Observable & Governed Weeks 1–10

Replace direct provider API key usage with a governed, observable gateway. Every AI call flows through one point. Finance and engineering get immediate cost visibility.

🔀 Gateway
Go reverse proxy. OpenAI-compatible /v1 endpoint. Sub-15ms overhead. Full SSE streaming support.
🔑 Virtual Keys
Create, revoke, rotate keys. Bind to users, teams, projects. Redis hot-cache for sub-ms resolution.
🌐 Providers
OpenAI, Anthropic, Azure OpenAI, Gemini. Extensible provider interface for custom models.
📋 Prompt Logs
Full request/response capture via Kafka → ClickHouse. S3 archival. Search in OpenSearch.
💲 Cost Tracking
Per-request token cost. Roll-up by user, model, day. Live dashboard. CSV export.
👤 User Management
RBAC: Admin / Developer / Analyst / Viewer. SSO via OAuth2 / SAML. Okta & Azure AD support.
V2 Governance — Finance & Security Controls Weeks 11–20

Give Finance the controls to manage AI spend and give Security the tools to prevent data leakage. Unlock enterprise budget cycles and compliance requirements.

💰 Budgets
Monthly/weekly/daily budget caps. Hard limits block requests. Soft threshold alerts at 80%. Per-team, per-project, per-user scopes.
🏢 Teams
Org → Department → Team → Project hierarchy. Inherit budget policies. Nested reporting.
📊 Chargebacks
Automated cost allocation reports. Per-team P&L attribution. CSV, PDF, API export for Workday/SAP.
🛡️ DLP
Presidio-based PII/secret detection. BLOCK / REDACT / ALERT / ALLOW policies. Custom entity patterns.
📜 Governance Policies
OPA + Rego: allowed models, time windows, geo restrictions, content filters. Version-controlled policy-as-code.
V3 Agent & Tool Governance Weeks 21–32

The agentic AI era demands new governance primitives. Every agent has an identity, an owner, and approved tools. No autonomous agent operates outside sanctioned boundaries.

🤖 Agent Registry
Register agents: name, owner, team, allowed tools, cost ceiling. Agent identity tied to a Virtual Key. Full audit trail.
🔧 Tool Governance
Approve/deny tool access per agent. Require human-in-the-loop for sensitive tools. Audit every tool invocation.
🗂️ AI Inventory
Auto-discover AI usage across the org. Surface shadow AI. Build a complete picture of all models in production.
⚡ Smart Routing
Route to cheapest/fastest model per SLA. Fallback chains on provider errors. A/B split for model evaluation.
V4 Intelligence & Optimization Weeks 33–44

Use the rich telemetry data from V1–V3 to build intelligent forecasts, semantic caches, and quality scorecards that make AI usage smarter and cheaper.

📈 Forecasting
30/60/90-day spend forecasts using Prophet/ARIMA on ClickHouse data. Anomaly detection. Budget overage alerts before they happen.
⚙️ Cost Optimization
Semantic prompt caching via pgvector (target: 20%+ cache hit rate). Model downgrade suggestions for eligible request patterns.
📝 Prompt Versioning
Version-controlled prompts with diff viewer. A/B test framework: compare cost, latency, and output quality across prompt versions.
🏆 AI Scorecards
Monthly quality + cost + latency health scores per team. Executive dashboards. Trend analysis. Benchmarked against org-wide averages.
